Gmail / Google Apps

To send email using Google Mail you will have to modify your outgoing server settings in the Mail Settings accordingly. This article will walk you through how to add an SMTP server to an Google Apps account, but you can also follow these steps to add an SMTP server to a general Gmail account.

Important If you use Gmail and do NOT have a Google Apps for Business or Google Apps for Education account, Gmail cannot be configured to use an external SMTP relay unless it is configured to work with an existing email server.

If you use Google Apps for Business or Education, then your only option is to configure an external SMTP route. This means that all of your outgoing email would be sent through SendGrid.

Google Apps allows users to define their own outbound SMTP relay configuration in their Gmail accounts. This still requires that you have a separate Mail Transfer Agent (MTA) configured to use SendGrid’s SMTP servers and you have accounts configured on that server through which people can send messages intended to consume credits.

Adding SMTP Settings

When making your decision, please know that Google applies the following limits:
Limit type Description Value
Messages per day Daily sending limit* 2000
Messages auto-forwarded Messages automatically forwarded from another account; not included in the daily sending limit 10,000
Auto-forward mail filters Account filters that automatically forward mail 20
Recipients per message Addresses in the To, Cc, and Bcc fields of a single email* 2000 (500 external)
Recipients per message (sent via SMTP by POP/IMAP users) Addresses in the To, Cc, and Bcc fields of a single email* 100
Total recipients per day Individual addresses count in every mail sent; 5 emails sent to 10 addresses count as 50 recipients* 10,000
External recipients per day Email addresses outside your primary domain. Includes domain aliases and alternate domains. 3000
Unique recipients per day Individual addresses count once a day; 5 emails sent to 10 addresses count as 10 unique recipients* 3000 (2000 external)

Enabling External SMTP Servers

If you’re using Google Apps, you have to enable your domain to allow users to send to an external SMTP server. This setting is located under Settings > Email and requires administrator rights to enable.

  • If you want all outgoing mail from your domain to go through SendGrid, add smtp.sendgrid.net to the Outbound Gateway setting field. If you choose to do this, this is the only step, and you’re done! What happens with this option defined is that all outgoing mail is sent to the gateway server instead of using Google’s SMTP servers.
  • If you only want to send some email through SendGrid, check the box in the Outbound Relay setting. What this does is allow users to configure their own SMTP server when adding a Send As account. This does not enable an administrator feature. This allows individual users to setup individual accounts that are external to the domain.

Info The Outbound Relay option is an administrative control that is only available in Google Apps accounts. Gmail-only users are able to use this feature without administrative management.

Info For the purposes of this guide, we will assume the Outbound relay setting was checked, and the Outbound gateway was left blank.

Adding an Alias

Once the Outbound relay setting has been turned on, you need to login to the account you want to send from, and go to Mail Settings > Accounts, and select “Add another email address you own”. Here, you will put in the address you wish to be sending from. You can also specify the reply-to address here. If you do not, any replies to messages sent under this alias will go to the alias’ account.

Info Creating a Send Mail As account does not deliver mail sent to that address into your Gmail inbox. You either need to configure that account to forward all or select mail to your Gmail address or use Gmail’s POP3 mail checking functionality, located conveniently below the Send Mail As feature.

The next step will ask you whether you want to use your default SMTP settings, or the other domain. Select the other domain (second radio button), and put in the following details:

  • SMTP Server: smtp.sendgrid.net
  • Port: Leave this option to the default of 587.
  • Username: SendGrid account username
  • Password: SendGrid account password
  • Encryption: Either option is acceptable.

Sending as the Alias

As you’re sending a message, you should now see a From: field as a dropdown. If you select the Alias you’ve set up with the SendGrid SMTP, it will be sent through our servers, with all apps applied as needed.
